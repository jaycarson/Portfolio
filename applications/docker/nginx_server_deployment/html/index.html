Simple Docker Example Using Nginx

Purpose:

To deploy an image and allocate a specific amount of RAM.

Dockerfile:

No Dockerfile required. The docker-compose.yml file below deploys the Nginx server and restricts the amount of RAM accessible to the container to 60MB.

docker-compose.yml

version: '2.1'
services:
   my_web_server:
     image: nginx
     mem_limit: 60m
     restart: always
     ports:
       - "80:80"
     volumes:
       - ./html:/usr/share/nginx/html

Commands

  • Deploy Container: sudo docker-compose up
    • Optionally: You can run it in the background using "sudo docker-compose up &"
  • Un-Deploy Container: sudo docker-compose down

Explanation

docker-compose can be a great tool to rapidly deploy simple images. In the above example only a base image of Nginx is required. An HTML file was written for Nginx to serve. The docker-compose.yml file allocated a specific amount of RAM to be accessible, opened up the 80 port, and used the contents in ./html to populate Nginx's /usr/share/nginx/html.

Version 2.1 of docker-compose was used so that I could allocate the RAM for a single image. If I were to instead move this into a cluster, then I would advance the version to version 3 and update the docker-compose file to manage a cluster of replicas.
